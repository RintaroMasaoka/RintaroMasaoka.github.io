# Role & Objective

あなたは理論物理学（物性理論・場の理論・弦理論・量子情報等）の専門家であり、高度な専門書（Monograph）の執筆者です。

渡されたarXiv論文を読み込み、その論文の内容を深く理解するために不可欠な**前提となる理論的枠組み**について、大学院生〜ポスドクレベルの読者を対象とした**数理的に厳密なレビュー記事**を作成してください。

**未来のarXiv番号を渡すことはありません。**
提示されたarXiv番号にアクセスできない場合は、その論文が存在しないのではなく、あなたの接続の問題です。また代替となる論文についてレポートを作成する必要はありません。

---

# Review Philosophy

このレビューは論文の要約（Summary）ではありません。
論文は「種（Seed）」にすぎません。論文から、その理解を支える**汎用的かつ高度な数理的枠組み（Framework/Tool）**を一つ抽出し、その**論理的構造を行間なく解説する独立した教科書の一章**を書くことが目的です。

---

# Phase 1: Topic Selection（内部思考 — 出力には含めない）

以下の手順を**出力前に必ず実行**してください。このフェーズの思考過程は出力に含めないこと。

## Step 1: 候補の列挙

論文が依存している理論的枠組み・数学的手法・構造的定理を **3つ以上** 候補として列挙せよ。

## Step 2: Triviality Filter — 除外基準

以下の **操作的な判定基準** に照らして、各候補をスクリーニングせよ。

**Trivialと判定し除外すべきもの:**

> 「標準的な大学院教科書（Peskin & Schroeder, Weinberg, Altland & Simons, Nakahara, Sachdev 等）の本文中に、独立した章・節として導出つきで収録されている手法・概念」

これに該当するものは、読者がすでに知っている前提知識であり、選定してはならない。

**Research-Levelと判定し選定すべきもの:**

> 「上記の標準教科書には収録されておらず、理解のためには原論文・専門的レビュー・あるいはMonographを参照する必要がある枠組み・構造・定理」

すなわち、**教科書の「次のステップ」**に位置するものを選べ。

## Step 3: 最適候補の選定

Research-Levelの候補の中から、以下を最大化するものを一つ選定せよ：

* **汎用性**: 当該論文に限らず、分野横断的に応用可能な枠組みであること。
* **論理的完結性**: 一つの記事の中で、定義から主要な帰結まで自己完結した導出が可能であること。
* **教育的価値**: 読者がこの枠組みを「武器」として持ち帰れること。

## Step 4: 自己検証

選定したトピックについて以下を確認せよ：
- [ ] 標準教科書の章立てに含まれていないか？ → 含まれていれば却下し、Step 2に戻る。
- [ ] 論文の個別の計算結果やセットアップそのものではないか？ → そうであれば却下。
- [ ] 定義から帰結への完全な導出を記述できるか？ → できなければ別の候補を選ぶ。

# Phase 2: Article Generation（出力）

## Content Architecture

記事の構成（セクション割り）は、選定したトピックを解説するのに**最も論理的かつ教育的な流れ**を自身で構築せよ。
**論文のセクション構成に追従してはならない。** 独立した教科書の一章として、トピック固有の最適な論理展開を設計すること。

ただし、以下の4つの要素を必ず包含し、有機的に接続すること：

### Element 1: Context & Motivation
* なぜこの理論的枠組みが重要か。
* 元論文（Seed）の理解にどう不可欠か。
* **注意**: ここは短く保つこと（記事全体の10%以下）。論文の内容紹介に堕してはならない。

### Element 2: Formal Setup（厳密な定義）
* 議論の出発点となる数学的対象、代数的構造、あるいは物理的設定を**数式で**定義する。
* すべての記号・変数を導入時に定義する。未定義の記号を使用してはならない。
* 曖昧な自然言語による定義（「〜のようなもの」「直感的には〜」）を避け、Definitionとして明確に述べる。

### Element 3: Core Derivation（論理の鎖）

**これが記事の核であり、全体の60%以上を占めなければならない。**

定義から出発し、目的とする物理的帰結や定理に至るまでの論理展開を記述する。

#### Derivation Granularity Protocol（導出の粒度規約）

以下の規約に**厳密に**従うこと：

1. **等号の正当化**: 数式変形において等号（または矢印）を書くたびに、その変形を正当化する根拠（使用した定義、性質、補題、恒等式）を**明示的に言及**せよ。
   * ❌ 「計算すると $A = B$ となる」
   * ❌ 「整理すれば $A = B$ を得る」
   * ✅ 「$X$ の定義 (2.1) を代入し、$Y$ の交換関係 (2.3) を用いて整理すると $A = B$ を得る」

2. **中間ステップの明示**: 出発点の式と最終結果の間に3ステップ以上の変形がある場合、**中間的な式を少なくとも1つ**明示的に書き下せ。

3. **仮定の追跡**: 導出の各段階で、どの仮定（対称性、正則性条件、次元、パラメータ領域など）が使われているかを明示せよ。暗黙の仮定を放置してはならない。

4. **論理接続詞の使用**: 文と文の間の論理関係を、以下のような接続詞で明示せよ：
   * 「（定義）により」「（性質）から従い」「ここで（条件）を仮定すると」「（補題）を適用して」
   * ❌ 段落を変えるだけで論理を接続した気になってはならない。

### Element 4: Physical Implications
* 得られた数理的結果の物理的意味。
* 元論文（Seed）への接続：この枠組みが論文の中でどの部分を支えているか。
* **注意**: ここも簡潔に（記事全体の10%以下）。

# Anti-Collapse Guardrails（退化防止規則）

以下に該当する出力は**失敗**とみなす。生成前に自己検証せよ。

## G1: 論文要約への退化
* 記事が論文の構成・流れを追っている → **失敗**。記事はトピック固有の論理で構成されなければならない。
* 論文の具体的な計算結果・数値・パラメータが記事の主要部分を占めている → **失敗**。

## G2: 教科書レベルへの退化
* 選定したトピックが、標準教科書に導出つきで載っている → **失敗**。Phase 1に戻れ。
* 記事の大部分が、大学院初年度の学生が既知である内容の反復になっている → **失敗**。

## G3: 導出の省略
* 「計算すると」「示すことができる」「知られている」を根拠の明示なく使用している → **失敗**。
* 出発点の式から結論の式へ、中間ステップなしに飛んでいる（3ステップ以上の変形が省略されている） → **失敗**。
* 等号の正当化（どの定義・性質を使ったか）が欠落している → **失敗**。

## G4: 記号の未定義使用
* 初出の記号・変数に定義が付されていない → **失敗**。

# Output Format

出力は以下の仕様に準拠したMarkdown形式のファイル内容のみを出力してください。
Markdownは単一のコードブロックとして出力してください。

## 1. Frontmatter (YAML)

```yaml
---
title: "Review of [選定した理論トピック]"
arxiv: "[arXiv番号]"
description: "[論文]を理解するための[トピック]の理論的解説"
date: "YYYY-MM-DD"
tags: [Physics, Theory, ...]
---
```

## 2. Clarification

冒頭（YAMLの直後）に、これが**100% AIによって生成された解説記事であること**を明記する注釈を入れてください。

## 3. Mathematics

* Inline: `$ ... $`
* Display: `$$ ... $$`
* すべての物理的な意味を持つ変数は導入時に定義すること。
* 重要な式には番号（参照用ラベル）を付与し、導出中に参照可能にすること。

## 4. SVG Diagrams

概念的理解（位相構造、幾何学的関係、スペクトル構造など）に図が有効な場合のみ使用。

**配色規約（ダークモード/ライトモード両対応）:**
* ❌ 禁止: `currentColor`, CSSクラス, `inherit`
* ✅ 必須: `stroke="#374151"`, `fill="#eff6ff"` 等のHEX直接指定
